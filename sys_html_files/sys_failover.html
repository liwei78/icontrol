mgmt/tm/sys/failover

Redundant system failover state

REST Endpoints

Collection URI
/mgmt/tm/sys/failover
Collection Methods
OPTIONS, GET
Resource URI
/mgmt/tm/sys/failover/~[resource id]
Resource Methods
OPTIONS, GET, POST
Resource Natural Key

Properties

NameTypeDefault ValueRequiredAccessDescription
devicestring optionalreadSpecifies the name of the device that should become the active device for the traffic group or for all traffic groups.
noPersistboolean optionalreadDoes not persist the change in status of a unit or cluster. Valid only with offline status.
offlineboolean optionalreadChanges the status of a unit or cluster to Forced Offline. If persist or no-persist are not specified, the change in status will be persisted in-between system restarts.
onlineboolean optionalreadChanges the status of a unit or cluster from Forced Offline to either Active or Standby, depending upon the status of the other unit or cluster in a redundant pair.
persistboolean optionalreadPersists the change in status of a unit or cluster. Valid only with offline status.
standbyboolean optionalreadSpecifies that the active unit or cluster fails over to a Standby state, causing the standby unit or cluster to become Active.
trafficGroupstring optionalreadSpecifies the name of the traffic-group that the standby command refers to.
