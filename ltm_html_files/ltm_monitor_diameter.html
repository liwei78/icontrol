timeoutinteger optionalread/writeSpecifies the number of seconds the target has in which to respond to the monitor request. The default value is 31 seconds. If the target responds within the set time period, it is considered up. If the target does not respond within the set time period, it is considered down. Also, if the target responds with a RESET packet, the system immediately flags the target as down without waiting for the timeout interval to expire.
upIntervalinteger optionalread/writeSpecifies how often in seconds that the system issues the monitor check when the node is up. The default value is the same as the (down) interval.
