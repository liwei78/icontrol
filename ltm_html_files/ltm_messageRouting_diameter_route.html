mgmt/tm/ltm/messageRouting/diameter/route

Configures a static route for use in Diameter message routing.

REST Endpoints

Collection URI
/mgmt/tm/ltm/messageRouting/diameter/route
Collection Methods
OPTIONS, GET
Resource URI
/mgmt/tm/ltm/messageRouting/diameter/route/~[resource id]
Resource Methods
OPTIONS, GET, PUT, PATCH, DELETE, POST
Resource Natural Key

Properties

NameTypeDefault ValueRequiredAccessDescription
appServicestring optionalread/writeThe application service to which the object belongs.
applicationIdinteger optionalread/writeSpecifies the identifier matching the application-id in the Diameter message. A value of 0 matches every application-id.
descriptionstring optionalread/writeUser defined description.
destinationRealmstring optionalread/writeSpecifies the destination realm matching the Destination-Realm AVP value in the message. The blank default value routes all destination-realms.
originRealmstring optionalread/writeSpecifies the origin realm matching the Origin-Realm AVP value in the message. The blank default value routes all origin-realms.
tmPartitionstringCommonoptionalread/write
peerSelectionModestringsequentialoptionalread/writeSpecifies how the system selects a peer to handle the Diameter traffic for this static route. The default is Sequential. Options are 1) Ratio: Peer selection is based on the ratio that is set for each peer that is in the Selected list. 2) Sequential: Peer selection is based on the order of the peers in the Selected list.
virtualServerstring optionalread/writeSpecifies the virtual server from which the system receives client requests for this static route. If you do not select a virtual server, the system uses this static route to route Diameter messages originating from any client.

Associated Arrays and Lists

Property NameTypeDefault ValueRequiredAccessDescription
peersstring optionalread/writeSpecify which peers should handle the Diameter traffic for this static route from the list. The static route uses the peers in the order they are specified.
